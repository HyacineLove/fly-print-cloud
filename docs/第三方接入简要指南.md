# 第三方接入简要指南（局域网演示）

> **本文适用范围：同一局域网内的演示 / 首次联调（M0）。**  
> 不是公网生产部署手册。公网差异见 [`第三方接入-公网部署要点.md`](第三方接入-公网部署要点.md)。

按本文可在 **校园网 / 办公室局域网** 把 Compose + Demo + Edge 配到「能提交一单第三方打印」。  
手机、Edge、跑 Compose 的电脑须在同一二层/三层可达网段。

官方扫码实跑、安装包现场预演见发版 checklist（M0-4），本文不替代安装包交付说明。

---

## 0. 局域网 vs 公网（先看清）

| | **局域网演示（本文）** | **公网 / 对外部署** |
|--|------------------------|---------------------|
| 谁访问 Cloud | 同网段电脑 + 手机 | 互联网用户 / 反向代理域名 |
| 扫码 URL 怎么来 | Edge 用 `cloud.base_url` 拼相对 `/entry`；可把 localhost 改写成局域网 IP | 须稳定 HTTPS 域名；见公网要点文 |
| `.env` / Compose 默认密钥 | **可不建 `.env`**，直接 `compose up`；演示默认值仅限局域网 | **必须**自建 `.env` 并轮换全部密钥（含 `OAUTH_CLIENT_SECRET_ENCRYPTION_KEY`） |
| `.env` 里 `EXTERNAL_API_URL` / `EXTERNAL_ADMIN_URL` / `ALLOWED_ORIGINS` | **一般不用改**（示例已含常见内网 CORS 通配） | **要按域名改** |
| 认证 | Compose 默认 `builtin` 即可 | 通常 Keycloak / HTTPS；属后续加固（约 M2） |
| Demo `callback_base_url` | 容器内 `http://integration-demo:8080` | 公网可达的 HTTPS 回调地址 |
| 目标 | 当场跑通一单 | 可对外长期服务 |

**本指南只覆盖左列。** 误按公网方式去改一堆 EXTERNAL_*，对局域网演示没有帮助，还容易改错。

---

## 1. 边界（必读）

- **第三方不直连打印机。** 只与 FlyPrint Cloud 对话；真正预览与打印在 **Edge 一体机** 上完成。
- **须终端确认。** 用户在 Edge 上看预览、选参数并确认后，Cloud 才建打印任务。
- **Demo 不是生产对接方。** `livacloud-demo` / `/integration-demo/` 仅用于验证协议；正式第三方用同样入口与 HMAC，换自己的 `entry_url` 与密钥即可。

---

## 2. 演示环境就绪清单（局域网）

在干净机器上按顺序勾选。管理端本机访问可用：`http://127.0.0.1:8012`（端口以 `.env` 中 `HTTP_PORT` 为准）。

### 2.1 启动 Cloud（可零配置）

局域网演示可直接：

```powershell
cd fly-print-cloud
docker compose up --build -d
```

不必先复制或修改 `.env`。本机打开 `http://127.0.0.1:8012`，默认账号 **`admin` / `admin123`**。

需要改端口或密码时再 `Copy-Item .env.example .env` 后编辑。  
**不必**为局域网演示改 `EXTERNAL_API_URL` / `EXTERNAL_ADMIN_URL` / `ALLOWED_ORIGINS`。  
公网/生产须轮换全部示例密钥，见公网要点文。

扫码能不能被手机打开，靠的是 **Edge 的 `cloud.base_url`（及本机局域网 IP 改写）**，不是靠改 Cloud 的 `EXTERNAL_API_URL`。

### 2.2 启动 Compose 并做健康检查

```powershell
docker compose up --build -d
docker compose ps
```

在跑 Compose 的电脑上检查：

- [ ] `GET http://127.0.0.1:8012/health`
- [ ] `GET http://127.0.0.1:8012/api/v1/health`

管理端：`http://127.0.0.1:8012/`（用 `DEFAULT_ADMIN_PASSWORD` 登录）。  
Demo：`http://127.0.0.1:8012/integration-demo/`。

**禁止**在还要保留数据的环境执行 `docker compose down -v`。

### 2.3 管理端确认 provider：`livacloud-demo`

打开管理端 **集成提供方**，创建或核对 code 为 `livacloud-demo`。可参考 Demo setup 页 JSON 模板：

| 字段 | 局域网演示常用值 |
|------|------------------|
| `code` | `livacloud-demo` |
| `entry_url` | `http://<Cloud 局域网 IP>:8012/integration-demo/entry`（例：`http://192.168.1.10:8012/integration-demo/entry`；手机要能打开） |
| `callback_base_url` | `http://integration-demo:8080`（**Compose 容器互访**，不是手机地址；主机名带连字符合法） |
| `allowed_ip_cidrs` | `172.16.0.0/12`（按实际 Docker 网段调整；勿写成 `127.x`） |
| `allowed_file_hosts` | `integration-demo`（精确主机名，不要带 `http://`） |
| `allow_private_file_hosts` | **打开**（演示必须）；仍须命中文件主机白名单 |

**常见填错（会导致 invalid / 后续拉文件失败）：**

- `entry_url` 不要写成 `http://IP/integration-demo:8080`（端口应在 IP 后：`:8012`，路径是 `/integration-demo/entry`）
- `callback_base_url` 保持 `http://integration-demo:8080`，不要改成局域网 IP
- 出口 CIDR 用 `172.16.0.0/12` 这类，不是 `127.16.0.0/32`

创建后保存 **入站 / 出站 HMAC 密钥**（只显示一次，立刻复制）。

### 2.4 把 HMAC 粘贴到 Demo setup

1. 打开 `http://127.0.0.1:8012/integration-demo/setup`（本机即可）
2. 输入 `INTEGRATION_DEMO_ADMIN_PASSWORD`
3. 粘贴 **入站密钥**（Demo → FlyPrint）与 **出站密钥**（FlyPrint → Demo）
4. 保存成功后密钥不会再回显

### 2.5 Edge：局域网可达、激活、默认 IPP

- [ ] Edge `cloud.base_url` = **手机可达的 Cloud 地址**（推荐直接写 `http://192.168.x.x:8012`）。若仍写 localhost，Edge 出码时会尝试改写成局域网 IP，但不如直接写 IP 稳妥。
- [ ] 管理端生成激活码 → Edge 激活 → 节点在线。
- [ ] 默认 IPP 打印机可用，管理端可见。

完成后：Edge 出码 → **同一局域网手机**扫码 → 进第三方 Demo 或官方上传。

## 3. 接入说明（协议摘要）

### 3.1 入口链路

```text
Edge 出示二维码（URL = cloud.base_url + /entry?...）
  → 手机打开 Cloud /entry（签发 terminal_ticket）
  → 重定向到 provider entry_url
  → 第三方登录 / 选文件
  → HMAC POST print-requests
  → Cloud → Edge 预览 → 用户确认 → 打印 → callback
```

官方自助打印走上传页；**第三方只拿终端票据，不直连打印机。**

### 3.2 创建打印请求（HMAC）

```http
POST /api/v1/integrations/{code}/print-requests
```

- `{code}` 示例：`livacloud-demo`
- 鉴权：HMAC 头（如 `X-FP-Client`、`X-FP-Signature`、`X-FP-Timestamp`、`X-FP-Nonce`）

### 3.3 文件主机白名单

- `allowed_file_hosts` 为精确主机名；演示配合 Compose 服务名 `integration-demo`。
- `allow_private_file_hosts=true` 仍不是全局私网放行。

### 3.4 状态与 callback

提交 → 终端预览确认 → 打印 → 终态回传 → 按配置 callback。Demo 会轮询到完成态。

### 3.5 时效

| 项 | 约略时长 |
|----|----------|
| 终端票据 | 约 **5 分钟** |
| Demo 文件 URL | 约 **10 分钟** |

---

## 4. Demo 逐步操作（局域网串起来）

1. 完成第 2 节。  
2. 确认手机能打开 `http://<Cloud 局域网 IP>:8012/health`。  
3. Edge 出码 → 手机扫码 → Demo 模拟 SSO → 选 PDF 提交。  
4. 回 **Edge 屏幕** 确认打印。  
5. Demo 页看到完成态（如 `completed`）。

---

## 5. 排障短表（局域网）

| 现象 | 优先检查 |
|------|----------|
| 手机扫码打不开 | 是否同网段；Edge `cloud.base_url`；防火墙是否放行 8012 |
| Demo 提交 401 | setup 密钥是否与管理端一致 |
| 文件主机拒绝 | `allowed_file_hosts` / `allow_private_file_hosts` |
| 票据无效 | 是否超过约 5 分钟；重新扫码 |
| `/health` 不通 | `docker compose ps` / logs |
