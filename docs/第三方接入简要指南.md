# 第三方接入简要指南

面向演示与首次联调：按本文可把 Compose + Demo + Edge 配到「能提交一单第三方打印」。  
官方扫码打印、完整现场预演见发版 checklist（M0-4），本文不替代安装包交付说明。

---

## 1. 边界（必读）

- **第三方不直连打印机。** 只与 FlyPrint Cloud 对话；真正预览与打印在 **Edge 一体机** 上完成。
- **须终端确认。** 用户在 Edge 上看预览、选参数并确认后，Cloud 才建打印任务。
- **Demo 不是生产对接方。** `livacloud-demo` / `/integration-demo/` 仅用于验证协议；正式第三方用同样入口与 HMAC，换自己的 `entry_url` 与密钥即可。

---

## 2. 演示环境就绪清单（M0-2）

在干净机器上按顺序勾选。默认入口：`http://127.0.0.1:8012`（端口以 `.env` 中 `HTTP_PORT` 为准）。

### 2.1 准备 `.env`

```powershell
cd fly-print-cloud
Copy-Item .env.example .env
```

首次启动前至少改掉示例值：

| 变量 | 说明 |
|------|------|
| `POSTGRES_PASSWORD` | 数据库密码 |
| `MINIO_ACCESS_KEY` / `MINIO_SECRET_KEY` | 对象存储 |
| `DEFAULT_ADMIN_PASSWORD` | 管理端登录（默认示例 `admin123`，务必改） |
| `OAUTH2_JWT_SIGNING_SECRET` | 内置认证 JWT |
| `FILE_ACCESS_SECRET` | 文件访问签名 |
| `REDIS_PASSWORD` | Redis |
| `INTEGRATION_DEMO_ADMIN_PASSWORD` | Demo setup 页密码（默认示例 `demo123`） |
| `OAUTH_CLIENT_SECRET_ENCRYPTION_KEY` | Base64 的 32 字节密钥（可用 `openssl rand -base64 32`） |

局域网给手机扫码时，把 `EXTERNAL_API_URL` / `EXTERNAL_ADMIN_URL` / `ALLOWED_ORIGINS` 改成 **手机能访问的主机名或局域网 IP**（不要只留 `localhost`）。

### 2.2 启动 Compose 并做健康检查

```powershell
docker compose up --build -d
docker compose ps
```

浏览器或 curl 检查：

- [ ] `GET http://127.0.0.1:8012/health`
- [ ] `GET http://127.0.0.1:8012/api/v1/health`

管理端：`http://127.0.0.1:8012/`（用 `DEFAULT_ADMIN_PASSWORD` 登录）。  
Demo 入口：`http://127.0.0.1:8012/integration-demo/`。

**禁止**在还要保留数据的环境执行 `docker compose down -v`（会删库与文件卷）。

### 2.3 管理端确认 provider：`livacloud-demo`

打开管理端 **集成提供方**（Integration Providers），创建或核对 code 为 `livacloud-demo` 的提供方。可参考 Demo setup 页上的 JSON 模板，关键字段：

| 字段 | 演示常用值 |
|------|------------|
| `code` | `livacloud-demo` |
| `entry_url` | `http://<可达主机>:8012/integration-demo/entry` |
| `callback_base_url` | Compose 内回调用 `http://integration-demo:8080`（容器互访） |
| `allowed_ip_cidrs` | 如 `172.16.0.0/12`（按实际 Docker 网段调整） |
| `allowed_file_hosts` | `integration-demo`（精确主机名） |
| `allow_private_file_hosts` | 演示开 `true`；**不是**全局私网放行，仍须命中白名单主机 |

创建后保存 **入站 / 出站 HMAC 密钥**（只显示一次，立刻复制）。

### 2.4 把 HMAC 粘贴到 Demo setup

1. 打开 `http://<主机>:8012/integration-demo/setup`
2. 输入 `INTEGRATION_DEMO_ADMIN_PASSWORD`
3. 粘贴 **入站密钥**（Demo → FlyPrint）与 **出站密钥**（FlyPrint → Demo）
4. 保存成功后密钥不会再回显

未配置密钥时，Demo 提交打印会失败。

### 2.5 Edge：局域网地址、激活、默认 IPP

- [ ] Edge 配置 `cloud.base_url` = **手机可达的局域网 IP + 端口**（例如 `http://192.168.x.x:8012`）。**禁止**给手机扫码场景使用 `localhost` / `127.0.0.1`。
- [ ] 在管理端生成激活码，Edge 完成激活，节点在线。
- [ ] 发现并设好默认 IPP 打印机，管理端可见且可用。

完成后应能：Edge 出码 → 手机打开 → 进入第三方 Demo（或官方上传）流程。

## 3. 接入说明（M0-3）

### 3.1 入口链路

```text
Edge 出示二维码
  → 用户打开 Cloud /entry（校验后签发 terminal_ticket）
  → 重定向到 provider 的 entry_url（携带 terminal_ticket）
  → 第三方完成登录 / 选文件
  → 第三方 HMAC 调用 print-requests
  → Cloud 向 Edge 下发预览
  → 用户在终端确认 → 打印 → 状态回传与 callback
```

官方自助打印走上传页；**第三方只拿终端票据，不直连打印机。**

### 3.2 创建打印请求（HMAC）

```http
POST /api/v1/integrations/{code}/print-requests
```

- `{code}` 示例：`livacloud-demo`
- 请求体含 `terminal_ticket`、外部订单/用户信息、文件 URL 等（以 API / Demo 实现为准）
- 鉴权：HMAC 头（如 `X-FP-Client`、`X-FP-Signature`、`X-FP-Timestamp`、`X-FP-Nonce`），密钥为管理端下发的入站密钥

Demo 内部即调用上述接口；自建第三方时复用同一路径与签名约定。

### 3.3 文件主机白名单

- Cloud 拉取第三方文件前检查 `allowed_file_hosts`（**精确主机名**）。
- `allow_private_file_hosts=true` 时仍只允许白名单主机，并拒绝环回等危险地址。
- 演示文件由 `integration-demo` 提供时，主机名须配成 `integration-demo`（与 Compose 服务名一致）。

### 3.4 状态与 callback

典型路径（简化）：

1. 第三方提交成功 → 请求进入等待终端 / 预览绑定  
2. Edge 展示预览 → 用户确认参数  
3. Cloud 建 `print_job` → Edge 打印  
4. 终态（如 `completed` / `failed` / …）回传 Cloud，并按 provider 配置 **callback** 通知第三方  

Demo 会轮询订单状态直到看到完成类结果。中途重启 Edge 时，已合入的恢复逻辑会尽量带上终端上下文，避免卡在 `dispatched` / `printing`（完整验证属预演项）。

### 3.5 时效

| 项 | 约略时长 | 过期表现 |
|----|----------|----------|
| 终端票据 `terminal_ticket` | 约 **5 分钟** | 需回终端重新扫码 |
| Demo 提供的文件 URL | 约 **10 分钟** | 提交或拉文件失败，需重新选文件提交 |

## 4. Demo 逐步操作（串起来）

1. 完成第 2 节清单（Compose、provider、HMAC、Edge 在线 + IPP）。  
2. 手机与 Edge / Cloud 同一局域网；Edge `base_url` 为局域网 IP。  
3. 在 Edge 选择第三方 / 出示二维码 → 手机扫码。  
4. 进入 Demo → 模拟 SSO → 在文档页选 PDF → 提交。  
5. 回到 **Edge 屏幕** 预览并确认打印。  
6. 在 Demo 页看到订单进入完成态（如 `completed`）。  

官方路径对照：扫码后走上传页上传文件，同样在终端确认后打印（不经 HMAC）。

## 5. 排障短表

| 现象 | 优先检查 |
|------|----------|
| 手机扫码打不开 / 一直转圈 | Edge `cloud.base_url` 是否 localhost；手机能否访问该 IP:8012；防火墙 |
| Demo 提交 401 / 签名失败 | setup 是否已贴对入站密钥；管理端是否轮换过密钥未同步到 Demo |
| 文件拉取失败 / 主机拒绝 | `allowed_file_hosts`、`allow_private_file_hosts`；Demo 是否用 `integration-demo` 主机名 |
| 提示票据无效或丢失 | 是否超过约 5 分钟；是否清了浏览器会话；回终端重新扫码 |
| 管理端无节点 / 无法预览 | Edge 是否已激活在线；打印机是否默认 IPP 且可用 |
| `/health` 不通 | `docker compose ps`；`docker compose logs -f api nginx` |
