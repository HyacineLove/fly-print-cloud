# 第三方接入：公网 / 对外部署要点

> **本文不是 M0 演示主路径。**  
> 局域网当场跑通请用 [`第三方接入简要指南.md`](第三方接入简要指南.md)。  
> 下文只说明：**公网与局域网差在哪里**，避免把两套配置搅在一起。

完整公网加固（HTTPS 强制、Keycloak、泄露审查等）按路线图约在 **M2+**，本文不代替那一套门禁。

---

## 1. 什么时候才需要看本文

同时满足或接近时：

- 手机 / 用户浏览器 **不在** Cloud 所在局域网；
- 或必须用 **公网域名 + HTTPS** 访问管理端 / `/entry` / 第三方 `entry_url`；
- 或第三方回调地址必须是公网可达 URL。

仅在教室/展厅同一 Wi‑Fi 下演示 → **只看局域网指南即可**，不要按本文改 `EXTERNAL_*`。

---

## 2. 与局域网演示的关键差异

| 项 | 局域网演示 | 公网 / 对外 |
|----|------------|-------------|
| 扫码拼 URL | Edge `cloud.base_url`（常为局域网 IP）；Cloud 回相对 `/entry` | `cloud.base_url` 应为 **https://你的域名**；用户手机必须解析并访问该域名 |
| `EXTERNAL_API_URL` | 可保持示例 `http://localhost:8012` | 改为对外 API 根 URL（与真实访问一致） |
| `EXTERNAL_ADMIN_URL` / `ADMIN_CONSOLE_URL` | 可保持示例 | 改为对外管理端 URL |
| `ALLOWED_ORIGINS` | 示例已含常见内网通配，通常不动 | 改为真实前端 Origin（具体域名，慎用过宽通配） |
| `OAUTH2_MODE` | `builtin` 足够演示 | 生产倾向 `keycloak` + 正确回调 URI |
| Provider `entry_url` | `http://局域网IP:8012/integration-demo/entry` | `https://域名/.../entry` |
| Provider `callback_base_url` | 容器名 `http://integration-demo:8080` | 公网可达的 HTTPS 回调基址（第三方能被 Cloud 调到） |
| TLS / 证书 / 反代 | 本机 HTTP 即可 | 须在网关终止 TLS；Cookie/CORS/重定向与域名一致 |
| 安全预期 | 演示网、默认可改密钥 | 须过公网加固清单后再接真流量 |

协议本身（HMAC、`print-requests`、终端确认、不直连打印机）**两边相同**；变的是 **URL 可达性与安全基线**。

---

## 3. 公网最小检查清单（提纲）

在已有局域网跑通经验的前提下，对外前至少确认：

1. [ ] 域名 DNS 指向入口；全站 HTTPS 有效  
2. [ ] `.env` 中 `EXTERNAL_API_URL`、`ADMIN_CONSOLE_URL`、`ALLOWED_ORIGINS`、OAuth 回调与域名一致  
3. [ ] Edge `cloud.base_url` 为公网 HTTPS 根，且现场能访问  
4. [ ] Provider `entry_url` / 回调地址均为公网 HTTPS  
5. [ ] 默认管理员密码与所有示例密钥已轮换；不把 Demo 密钥当生产密钥  
6. [ ] 按阶段计划完成公网加固项后再接非演示流量  

细节以当时 `README.md`、安全清单与里程碑（M2+）为准。
