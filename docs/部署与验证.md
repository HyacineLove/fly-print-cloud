# FlyPrint 部署与验证

> **读者：** 负责把 Cloud + Edge 落到可验收环境的人 / Agent。  
> **默认按互相可达的正式部署写**（通常即 **HTTPS 公网** + 受信证书）。局域网 Compose、无 TLS、自带 Demo 见文末 **§6 特例**。  
> **产品行为与边界：** [`系统说明.md`](系统说明.md)。  
> **第三方对接：** [`第三方接入指南.md`](第三方接入指南.md)（部署本文不替代其契约）。

**适用版本：** Edge 安装包以 `fly-print-edge/installer.iss` / `edge_node_info.py` 当前版本为准（产物在 `dist/`，不入库）；Cloud 以当前 `docker-compose.yml` 为准。

---

## 0. 协作模型（期望终态）

```text
准备域名与受信证书 + 改完默认密钥
  → 部署 Cloud（TLS 终止后对外仅 https）
  → 安装 Edge，cloud.base_url = https://域名
  → 激活节点、设默认 IPP
  → 官方扫码打印至 completed
  →（可选）按第三方指南联调一单
```

**边界（不可违反）：**

- 公网 **禁止**继续使用演示默认密码/密钥。  
- Edge `cloud.base_url` 对手机必须可达；**禁止**给手机链路用 `localhost`。  
- HTTPS **禁止**依赖「`https://localhost` + 出码改写局域网 IP」（证书主机名会失败）。  
- 需保留数据时 **禁止** `docker compose down -v`。

**仓库依据（程序内）：** Compose `HTTP_PORT`/`FLY_PRINT_*`（`docker-compose.yml`、`.env.example`）；Edge `cloud.base_url`（`config.example.json`）；kiosk（`launcher.py`）；健康检查（`api/cmd/server/main.go`）。

## 1. 一次准备清单（部署完成定义）

双方 / 运维只准备下面这些，再开始装机。

### 1.1 公网 Cloud 需要

| 项 | 说明 | 示例形态 |
|----|------|----------|
| 域名 | 手机与 Edge 都解析到 Cloud 入口 | `print.example.com` |
| 受信 TLS 证书 | 系统信任链；**不支持自签**作出站信任 | Let’s Encrypt / 云厂商证书 |
| TLS 终止方式 | 推荐在 Compose **前方**用反代/负载均衡终止 HTTPS，回源到本机 `HTTP_PORT`（默认容器 80→主机 8012）。仓库 nginx 的 `443`/ssl 卷默认注释，未开箱即用 | 反代 → `http://127.0.0.1:8012` |
| 强密钥 | 管理员密码、JWT、文件访问、MinIO、OAuth 相关 | 写入 `.env`，**勿提交仓库** |
| 防火墙 | 对公网开放 **443**（及你放行的管理来源）；数据库/MinIO/Redis **不对公网** | — |

`.env` 中与对外 URL 相关的项（见 `.env.example` 注释）建议与域名对齐，例如：

- `EXTERNAL_API_URL=https://print.example.com`
- `EXTERNAL_ADMIN_URL` / `ADMIN_CONSOLE_URL` / `OAUTH2_REDIRECT_URI`
- `ALLOWED_ORIGINS` 含 `https://print.example.com`

扫码可达性最终仍以 Edge 的 **`cloud.base_url`** 为准。

### 1.2 Edge 一体机需要

| 项 | 说明 |
|----|------|
| 安装包 | `flyprint-edge-setup-<版本>.exe`（`PrivilegesRequired=lowest`，不强制管理员安装） |
| `cloud.base_url` | `https://print.example.com`（无尾路径也可；与手机打开的根一致） |
| 本机 Edge 浏览器 | 启动器解析 `msedge.exe` |
| IPP 打印机 | 与一体机可达；将在 Edge 管理端设为默认 |
| Windows 手势等 | **程序外**，见 §4 |

### 1.3 验收账号（部署后立即改）

| 用途 | 演示默认（仅内网临时） | 公网 |
|------|------------------------|------|
| Cloud 管理端 | `admin` / `admin123`（`DEFAULT_ADMIN_PASSWORD`） | **必须修改** |

Edge 本机管理页默认仅回环访问，**没有**单独的「Edge 本机管理员密码」（见 [`系统说明.md`](系统说明.md) §3.3）。

## 2. 端到端步骤（HTTPS 公网主路径）

### 2.1 部署 Cloud

```powershell
cd fly-print-cloud
Copy-Item .env.example .env   # 公网必须改密钥与 EXTERNAL_* / ALLOWED_ORIGINS
# 编辑 .env 后：
docker compose up --build -d
docker compose ps
```

检查（经 TLS 反代后用 https；若暂在主机上直连 Compose 入口则用 http://127.0.0.1:8012）：

- [ ] `GET /health`、`GET /api/v1/health` 返回正常  
- [ ] 管理端可登录（已改密）  
- [ ] 手机流量下可打开 `https://域名`（证书无告警）

Compose 会顺带起 `integration-demo`；**官方打印不依赖它**，可不配置 Provider。

更新代码后：

```powershell
docker compose up --build -d
# 仅 API 时：docker compose up --build -d api
```

### 2.2 安装与激活 Edge

1. 安装当前版本 `flyprint-edge-setup-*.exe`。  
2. 打开本机管理页（启动器托盘 **「打开管理页」** 或本机 `http://127.0.0.1:7860`），设置：  
   - `cloud.base_url` = `https://print.example.com`  
3. Cloud 管理端生成激活码 → Edge 激活 → 节点显示在线。  
4. 刷新 IPP 打印机并 **设为默认** → Cloud 管理端可见。

Edge↔Cloud：REST 用 https，WS 自动 **wss**（`url_scheme.py`）。

### 2.3 官方扫码打印（主验收）

1. Edge 用户页出示二维码。  
2. 手机扫码 → `/entry` → **官方打印** → 上传 PDF。  
3. 回一体机预览 → **立刻打印**。  
4. Cloud 管理端任务到 **`completed`**。

**现场纪律（程序行为）：** 进门后二维码遮挡（占用）；点刷新作废手机旧会话；打印中不可中断回扫码；预览「返回」直接回扫码页。

### 2.4（可选）第三方一单

按 [`第三方接入指南.md`](第三方接入指南.md) 交换清单并启用 Provider；联调检查单见该文 §8。  
公网时 `entry_url` / `callback_base_url` / `file.url` 使用 https 与可达主机；需 Redis（Compose 默认已带）。

## 3. 最小验证检查单

Cloud：

- [ ] https 域名可打开管理端；默认密码已改  
- [ ] `/health`、`/api/v1/health`  
- [ ] 未对公网暴露 Postgres / Redis / MinIO 控制口  

Edge：

- [ ] 节点在线；默认 IPP 可用  
- [ ] `cloud.base_url` 为 https 域名（非 localhost）  

闭环：

- [ ] 官方打印一单 → 任务 `completed`  
- [ ]（可选）第三方或 Demo 一单 → `completed`  

程序外：

- [ ] §4 手势 / 误触项已按现场收紧

## 4. 程序外设置（一体机 Windows）

安装包只能拉起 Edge kiosk（`launcher.py`），**不能**关闭系统边缘滑动。建议：

| 类 | 建议 |
|----|------|
| 边缘滑动 / 多指手势 / 平板模式 | 关闭或固定桌面，避免滑出任务视图 |
| 任务栏 / 通知 | 自动隐藏；营业时段减少横幅 |
| 休眠与更新 | 关休眠；避免营业中强制重启 |
| 账户 | 营业账户非管理员；机箱上锁 |
| 网络 | 一体机固定 IP 或 DHCP 保留；手机须能访问 Cloud，不必访问 Edge 回环 |

可选注册表示例（因 OEM 而异，改前备份）：

```powershell
New-Item -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\ImmersiveShell" -Force | Out-Null
Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\ImmersiveShell" `
  -Name "EdgeSwipeEnabled" -Type DWord -Value 0 -Force
```

## 5. 排障速查（主路径）

| 现象 | 先查 |
|------|------|
| 证书告警 / 手机打不开 | 域名、受信证书、反代是否指到 Compose 入口 |
| 扫码后空白或错误 | `cloud.base_url` 是否 https 域名；是否误用 localhost |
| Edge 不在线 | 激活码；出站 443/wss 是否被拦；`cloud.base_url` |
| 有预览无出纸 / `ipp_*` | Edge↔打印机 IPP、默认机、耗材；非 Cloud 直连 |
| 卡在等待终端确认 | Edge 是否在线；是否占用遮挡未进预览；Cloud 是否下发了预览（日志「收到文件预览请求」） |
| 管理端英文 `invalid_grant` 等 | 已映射常见文案（`admin/src/utils/mapApiError.ts`）；漏网记原文 |
| 打印任务打印机名为 `-` | Cloud 需含 `NULLIF(display_name,'')` 回退本名的版本 |
| 数据没了 | 是否误执行 `docker compose down -v` |

```powershell
docker compose logs -f api nginx
```

## 6. 特例（非公网 HTTPS 默认时）

协议与产品行为不变；变的是 URL、端口与 CIDR。

### 6.1 局域网 Compose（http + 端口 8012）

```powershell
cd fly-print-cloud
docker compose up --build -d
```

- 入口：`http://<Cloud局域网IP>:8012`  
- 默认账号：`admin` / `admin123`（仅临时演示）  
- Edge `cloud.base_url`：`http://<Cloud局域网IP>:8012`  
- 一般**不必**为启动去改 `EXTERNAL_API_URL`；扫码靠 Edge 的 base_url  
- 停机保留数据：`docker compose down`（不要 `-v`）

### 6.2 少起 / 不跑 Demo 容器

官方打印不创建 Provider 即可。可选：

```powershell
docker compose up --build -d --scale integration-demo=0
```

### 6.3 Compose 同机第三方 Demo

| 字段 | 常用值 |
|------|--------|
| code | `livacloud-demo` |
| entry_url | `http://<LAN-IP>:8012/integration-demo/entry`（手机） |
| callback_base_url | `http://integration-demo:8080`（Cloud→Demo） |
| allowed_ip_cidrs | 常见 `172.16.0.0/12`（**勿写 127.x**） |
| allowed_file_hosts | `integration-demo`；`allow_private_file_hosts` 开 |

HMAC 粘贴：`/integration-demo/setup`（默认密码 `demo123`）。细节与契约仍以 [`第三方接入指南.md`](第三方接入指南.md) §7 为准。

### 6.4 只部署 Cloud（暂无 Edge）

可验证管理端与健康检查；**不能**完成出码、预览确认、真实打印。

### 6.5 外接 PostgreSQL / Redis / MinIO

默认 Compose 将 API 指到服务名 `postgres` / `redis` / `minio`。外接时改 `docker-compose.yml` 中 `FLY_PRINT_DATABASE_*`、`.env` 的 `INTEGRATION_REDIS_URL` / `MINIO_*` 等，并确保 API 容器网络可达；仅改 `.env` 里未映射进 compose 的 host **不够**。

### 6.6 本地拆开开发

- API：`api/` 下 `go run`，读 `config.yaml` / `FLY_PRINT_*`  
- Admin：`npm start`  
- 见 Cloud [`README.md`](../README.md)「独立开发」  
- Edge 仍只配可达的 `cloud.base_url`

### 6.7 交付物更新约定

- Cloud 有改动：整轮结束后 `docker compose up --build -d`  
- Edge 有运行时改动：bump 版本并 `release/build_installer.py` 打安装包  
- 见双仓 `AGENTS.md`「交付收口」
