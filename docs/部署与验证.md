# 飞印（FlyPrint）部署与验证

| 属性 | 说明 |
|------|------|
| 文档类型 | 部署、配置与验证指南 |
| 适用产品 | 飞印云打印平台（Cloud + Edge） |
| 默认场景 | 公网 HTTPS：受信证书 + TLS 终止后对外提供服务 |
| 相关文档 | [系统说明](系统说明.md) · [第三方接入指南](第三方接入指南.md) |

本文描述如何将 Cloud 与 Edge 部署到可对外服务的环境，并完成最小闭环验证。产品行为与安全边界见 [系统说明](系统说明.md)；第三方接口契约见 [第三方接入指南](第三方接入指南.md)。

**版本说明：** Edge 以安装包版本为准；Cloud 以当前 Compose 与镜像构建为准。

---

## 1. 部署总览

推荐部署顺序：

1. 准备域名、受信 TLS 证书，并完成密钥与对外 URL 配置  
2. 部署 Cloud（建议在 Compose 前方由反代或负载均衡终止 HTTPS）  
3. 安装 Edge，将 `cloud.base_url` 指向对外 HTTPS 根地址  
4. 激活节点并设置默认 IPP 打印机  
5. 完成官方扫码打印闭环（可选再联调第三方）

### 1.1 强制要求

| 要求 | 说明 |
|------|------|
| 密钥 | 公网环境不得继续使用演示默认管理员密码与各类密钥 |
| 可达性 | Edge 的 `cloud.base_url` 须为手机与一体机均可访问的地址；不得对手机链路使用 `localhost` |
| HTTPS | 不得依赖「`https://localhost` + 出码改写局域网 IP」；证书主机名须与访问域名一致 |
| 数据 | 需保留数据时，禁止对 Compose 执行带数据卷删除的销毁操作（例如 `docker compose down -v`） |

## 2. 部署前准备

### 2.1 Cloud（公网）

| 项 | 说明 | 示例 |
|----|------|------|
| 域名 | 手机与 Edge 均解析到 Cloud 入口 | `print.example.com` |
| 受信 TLS 证书 | 系统信任链；不支持以自签证书作为出站默认可信 | 公共 CA / 云厂商证书 |
| TLS 终止 | 建议在 Compose **前方**终止 HTTPS，回源至本机 HTTP 入口（默认主机端口 **8012**） | 反代 → `http://127.0.0.1:8012` |
| 密钥与配置 | 管理员密码、JWT、文件访问、对象存储、OAuth 等相关项 | 写入部署环境配置，勿写入公开仓库 |
| 防火墙 | 对公网开放 **443**（及经授权的管理来源）；数据库、缓存、对象存储控制口不对公网开放 | — |

对外 URL 相关配置建议与域名保持一致，例如 API / 管理端根地址、OAuth 回调、允许的跨域来源等。扫码业务的最终可达性仍以 Edge 侧 **`cloud.base_url`** 为准。

### 2.2 环境变量（公网必改项）

以仓库根目录 `.env.example` 为模板复制为 `.env` 后编辑。下表为公网 HTTPS 部署时须重点核对的项；未列出的项可先沿用 Compose 默认，再按外接组件需要调整。

| 变量 | 用途 | 公网建议 |
|------|------|----------|
| `HTTP_PORT` | Compose 统一 HTTP 入口（主机端口） | 默认 `8012`；反代回源至此端口 |
| `EXTERNAL_API_URL` | 对外 API / 业务根地址 | `https://print.example.com` |
| `EXTERNAL_ADMIN_URL` | 对外管理端根地址 | 通常与上相同 |
| `ADMIN_CONSOLE_URL` | 管理端对外 URL | 与域名一致 |
| `ALLOWED_ORIGINS` | CORS 允许来源（CSV，支持有限通配） | 至少包含 `https://print.example.com` |
| `OAUTH2_REDIRECT_URI` | OAuth 回调（builtin 模式） | `https://print.example.com/auth/callback` |
| `COOKIE_SECURE` | Cookie 的 Secure 标志 | HTTPS 下设为 `true` |
| `DEFAULT_ADMIN_PASSWORD` | 首次创建的默认管理员密码 | 强密码；部署后立即登录修改 |
| `OAUTH2_JWT_SIGNING_SECRET` | 内置认证 JWT 签名密钥 | 随机强密钥（如 `openssl rand -hex 32`） |
| `FILE_ACCESS_SECRET` | 文件访问凭证签名密钥 | 随机强密钥 |
| `OAUTH_CLIENT_SECRET_ENCRYPTION_KEY` | OAuth 客户端密钥加密主密钥 | Base64 编码的 32 字节；丢失后须重置客户端密钥 |
| `POSTGRES_PASSWORD` | 数据库密码 | 强密码 |
| `REDIS_PASSWORD` | Redis 密码（集成 nonce 等依赖） | 强密码；勿沿用演示默认值 |
| `MINIO_ACCESS_KEY` / `MINIO_SECRET_KEY` | 内置对象存储凭证 | 生产环境轮换；外接存储时另配 `STORAGE_*` / `MINIO_*` |
| `TRUSTED_PROXY_CIDRS` | API 可信反向代理 CIDR（CSV） | 经 Nginx / 外层反代时填写其网段（如 `172.16.0.0/12`），以便集成 API 正确识别第三方真实出口 IP；详见 [第三方接入指南](第三方接入指南.md) §7.1 |

说明：

- `OAUTH2_MODE=builtin` 为 Compose 默认；改用外部 IdP 时再配置 `keycloak` 相关变量。  
- 仅修改未注入运行容器的配置通常不生效；变更后执行 `docker compose up --build -d`。  
- Edge 的 `cloud.base_url` **不是** `.env` 项，在一体机本机管理页配置，且须与上表对外根地址一致。

### 2.3 Edge（一体机）

| 项 | 说明 |
|----|------|
| 安装包 | Windows 安装程序（普通用户权限即可安装） |
| `cloud.base_url` | 与手机打开的 Cloud 根地址一致，例如 `https://print.example.com` |
| 浏览器 | 本机已安装 Microsoft Edge（启动器依赖其可执行文件） |
| 打印机 | 与一体机网络可达，支持 IPP；在 Edge 管理端设为默认 |
| 操作系统 | 建议按本文第 5 节收紧触控手势与误触相关系统设置 |

### 2.4 管理账号

| 用途 | 说明 |
|------|------|
| Cloud 管理端 | 演示环境可能存在默认账号；**公网部署后须立即修改密码** |
| Edge 本机管理 | 默认仅本机回环访问，**无独立本机管理员密码**（见 [系统说明](系统说明.md)） |

## 3. 部署步骤（HTTPS 公网）

### 3.1 部署 Cloud

```powershell
cd fly-print-cloud
Copy-Item .env.example .env
# 编辑 .env：修改密钥，并使 EXTERNAL_* / ALLOWED_ORIGINS 等与域名一致
docker compose up --build -d
docker compose ps
```

部署后检查（经 TLS 反代使用 HTTPS；若临时在主机上直连 Compose 入口，可使用 `http://127.0.0.1:8012`）：

1. `GET /health` 与 `GET /api/v1/health` 正常  
2. 管理端可登录，且已使用非默认密码  
3. 移动网络环境下可打开 `https://域名`，证书无告警  

Compose 可能同时启动集成演示容器。**官方打印不依赖该容器**，未配置第三方时即可忽略。

代码或配置更新后，在同一目录执行：

```powershell
docker compose up --build -d
```

### 3.2 安装与激活 Edge

1. 安装当前版本的 Edge 安装包。  
2. 打开本机管理页（系统托盘「打开管理页」，或本机 `http://127.0.0.1:7860`），设置 `cloud.base_url` 为 `https://print.example.com`。  
3. 在 Cloud 管理端生成激活码，于 Edge 完成激活，确认节点在线。  
4. 发现 IPP 打印机并设为默认；在 Cloud 管理端确认打印机可见。  

Edge 与 Cloud 之间：REST 使用 HTTPS 时，WebSocket 自动使用 **WSS**。

### 3.3 官方扫码打印（主验证）

1. Edge 用户界面出示二维码。  
2. 手机扫码进入入口页，选择官方打印并上传文件。  
3. 在一体机预览参数后确认打印。  
4. 在 Cloud 管理端确认任务状态为 **已完成（`completed`）**。  

进门后终端将遮挡二维码；刷新二维码会作废未完成的手机侧会话；打印过程中不可中断返回扫码页。

### 3.4 第三方联调（可选）

按 [第三方接入指南](第三方接入指南.md) 完成密钥交换与 Provider 配置。公网环境中，入口地址、回调基址与文件地址须使用可达主机与 HTTPS。集成能力依赖 Redis（Compose 默认已包含）。

## 4. 验证检查单

**Cloud**

- [ ] HTTPS 域名可打开管理端，默认密码已修改  
- [ ] 健康检查接口正常  
- [ ] 数据库、缓存、对象存储控制口未对公网暴露  

**Edge**

- [ ] 节点在线，默认 IPP 打印机可用  
- [ ] `cloud.base_url` 为 HTTPS 域名（非 localhost）  

**业务闭环**

- [ ] 官方扫码打印一单至完成  
- [ ] （可选）第三方或集成演示一单至完成  

**现场环境**

- [ ] 第 5 节中的系统手势与误触相关项已按现场要求配置

## 5. 一体机操作系统建议

安装包可将用户界面拉起为全屏 kiosk，但**不能**替代操作系统层的触控与手势策略。建议在一体机上收紧下列设置（具体路径因 Windows 版本与 OEM 定制而异）：

| 类别 | 建议 |
|------|------|
| 边缘滑动 / 多指手势 / 平板模式 | 关闭或固定为桌面模式，避免误滑出系统界面 |
| 任务栏与通知 | 自动隐藏任务栏；营业时段减少通知横幅 |
| 电源与更新 | 关闭休眠；避免营业时段强制重启更新 |
| 账户与物理安全 | 营业账户使用非管理员权限；机箱上锁 |
| 网络 | 一体机使用固定 IP 或 DHCP 保留；手机须能访问 Cloud，无需访问 Edge 回环地址 |

可选注册表示例（效果因系统版本而异，修改前请备份）：

```powershell
New-Item -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\ImmersiveShell" -Force | Out-Null
Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\ImmersiveShell" `
  -Name "EdgeSwipeEnabled" -Type DWord -Value 0 -Force
```

修改后建议注销或重启，并用触控从屏幕边缘滑动验证系统界面是否仍被唤起。

## 6. 常见问题

| 现象 | 排查要点 |
|------|----------|
| 证书告警或手机无法打开 | 域名解析、受信证书、反代是否指向 Compose HTTP 入口 |
| 扫码后空白或错误 | `cloud.base_url` 是否为正确的 HTTPS 域名；是否误用 localhost |
| Edge 显示离线 | 激活状态；出站 HTTPS / WSS 是否被拦截；`cloud.base_url` 是否正确 |
| 有预览无出纸 | Edge 与打印机之间的 IPP 连通性、默认打印机、耗材状态（非 Cloud 直连问题） |
| 手机侧长时间等待终端确认 | Edge 是否在线；是否停留在占用遮挡而未进入预览；Cloud 是否已下发预览 |
| 第三方集成 API 返回 403 | Provider `allowed_ip_cidrs` 是否包含第三方真实出口 IP；Cloud 是否配置 `TRUSTED_PROXY_CIDRS` 以在反代后解析真实 IP |
| 管理端提示英文授权错误 | 常见登录错误已映射为中文；若仍出现，请记录完整原文以便排查 |
| 任务列表打印机名称缺失 | 使用已包含「无显示名时回退设备本名」逻辑的 Cloud 版本 |
| 数据丢失 | 是否误执行了删除数据卷的 Compose 销毁命令 |

查看 Cloud 日志示例：

```powershell
docker compose logs -f api nginx
```

## 7. 其他部署形态

下列形态不改变产品协议与行为，仅调整地址、端口或配套组件。

### 7.1 局域网 Compose（HTTP）

适用于内网演示或联调：

```powershell
cd fly-print-cloud
docker compose up --build -d
```

- 入口：`http://<Cloud局域网IP>:8012`  
- Edge `cloud.base_url`：与上相同  
- 演示默认管理员账号仅可用于临时内网环境，公网必须修改  
- 停止服务并保留数据：`docker compose down`（不要附加删除数据卷的参数）

### 7.2 不启用集成演示容器

官方打印无需创建第三方 Provider。可选不启动演示容器：

```powershell
docker compose up --build -d --scale integration-demo=0
```

### 7.3 同机第三方演示联调

| 配置项 | 常用做法 |
|--------|----------|
| 接入编码 | **`livacloud-demo`**（与仓库 `integration-demo` 一致） |
| 入口 URL | `http://<Cloud局域网IP>:8012/integration-demo/entry`（手机可达） |
| 回调基址 | `http://integration-demo:8080`（Cloud 容器内服务名，**非**手机地址） |
| 文件主机 | `integration-demo`；`allow_private_file_hosts=true` |
| 源 IP 白名单 | `172.16.0.0/12`（覆盖 Docker 桥接；勿用 `127.0.0.0/8`） |
| 密钥配置页 | 对外 `/integration-demo/setup`（容器内为 `/setup`） |

创建 Provider 后，在 setup 页粘贴入站/出站 HMAC 密钥。契约与排障见 [第三方接入指南](第三方接入指南.md) §8。

### 7.4 仅部署 Cloud

可验证管理端与健康检查；在缺少 Edge 与打印机时，无法完成出码、预览确认与真实出纸。

### 7.5 外接数据库、缓存与对象存储

默认 Compose 将服务指向内置组件。外接时须同步调整服务连接配置，并保证 API 容器网络可达；仅修改未注入运行环境的配置项通常无效。

### 7.6 交付物更新

| 变更范围 | 建议操作 |
|----------|----------|
| Cloud | 整轮变更结束后重建并拉起 Compose |
| Edge 运行时 | 提升版本号并重新生成安装包后升级现场一体机 |
