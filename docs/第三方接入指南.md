# 第三方接入指南（对接契约）

> **读者：** 第三方业务系统的开发 / Agent（拿本文即可实现对接）。  
> **FlyPrint 运维：** 用对方交来的「一次交换清单」在管理端创建 Provider，互贴 HMAC，启用即可。  
> **默认按互相可达的正式对接写**（通常即公网或固定专线）；局域网 / Compose 自带 Demo 见文末 **§7 特例**。  
> **权威实现：** `api/internal/handlers/integration_*.go`、`api/internal/integration/*`；参考实现：`integration-demo/`。

---

## 0. 协作模型（期望终态）

```text
第三方按本文开发完成
  → 双方交换「一次必要信息」+ HMAC 密钥
  → FlyPrint 管理端创建/启用 Provider
  → 联调一单
  → 之后无额外协议沟通
```

**边界（不可违反）：**

- 第三方 **不**直连打印机，也 **不**直连 Edge。  
- 用户必须在 **Edge 一体机**上预览并确认后，才会真正打印。  
- 第三方只负责：入口页、选文件/下单、HMAC 调 Cloud、收状态回调。

**传输与 URL 约定（对接前必读）：**

- Provider 的 `entry_url`、`callback_base_url`，以及请求里的 `file.url`，支持 **`http` 与 `https`**（禁止带 userinfo）。正式对接推荐 https。  
- **不支持自签证书**：Cloud 出站拉文件 / 发回调使用系统默认信任链。  
- 启用 Provider 前，Cloud 侧必须已配置 **Redis**（nonce 防重放）。

## 1. 一次交换清单（对接完成定义）

双方只交换下面这些；**不要**再另发明口令流程。

### 1.1 第三方 → FlyPrint（创建 Provider 用）

| 字段 | 说明 | 示例形态 |
|------|------|----------|
| `code` | 稳定唯一 ID；路径与 `X-FP-Client` 用；`^[a-z][a-z0-9-]{1,62}$` | `acme-hr` |
| `display_name` | `/entry` 选择页按钮文案 | `ACME 人事打印` |
| `entry_url` | **用户手机浏览器**打开的入口（Cloud 会追加 `terminal_ticket`） | `http://third.example.com/flyprint/entry` |
| `callback_base_url` | **Cloud 出站**可达的回调基址；实际 POST 到 `{base}/api/print/callback` | `http://third.example.com` |
| `allowed_ip_cidrs` | 第三方调用 Cloud 集成 API 时的 **出口 IP/CIDR**（CSV） | `203.0.113.10/32` |
| `allowed_file_hosts` | `file.url` 允许的 **主机名**（CSV，无 scheme/路径） | `files.third.example.com` |
| `allow_private_file_hosts` | 文件主机解析到 RFC1918 时是否允许（仍须在白名单内） | 公网通常 `false` |
| `max_file_size` | 字节上限 | `10485760` |
| `allowed_mime_types` | MIME 白名单 CSV | `application/pdf` |

### 1.2 FlyPrint → 第三方

| 信息 | 说明 |
|------|------|
| Cloud 集成 API 根 | 如 `http://print.example.com`（实际部署根；下列路径挂在 `/api/v1/...`） |
| `code` | 与对方约定一致 |
| **入站 HMAC 密钥** `inbound_hmac_secret` | 第三方签「调用 Cloud」的请求（创建 Provider / 轮换时 **只显示一次**） |
| **出站 HMAC 密钥** `outbound_hmac_secret` | 第三方验「Cloud 回调」签名（同上，只显示一次） |

### 1.3 HMAC 互配（唯一密钥交流）

| 方向 | 密钥 | 谁签 | 谁验 |
|------|------|------|------|
| 第三方 → FlyPrint | **入站** inbound | 第三方 | Cloud |
| FlyPrint → 第三方 | **出站** outbound | Cloud | 第三方 |

轮换：管理端「轮换密钥」后双方同时更新；旧密钥立即失效。

## 2. 端到端流程

```text
用户在 Edge 扫码
  → 手机打开 Cloud GET /entry?...
  → Cloud 签发 terminal_ticket（约 5 分钟）
  → 用户选择你们的 display_name
  → Cloud 302/引导到：
       GET {entry_url}?terminal_ticket=<opaque>
  → 第三方：登录 / 选文件 / 确认业务单
  → 第三方：HMAC POST /api/v1/integrations/{code}/print-requests
       （body 带同一 terminal_ticket + 文件元数据）
  → Cloud：拉取 file.url → 下发 Edge 预览
  → 用户在 Edge 确认参数
  → Cloud：建打印任务 → Edge IPP 打印
  → Cloud：HMAC POST {callback_base_url}/api/print/callback（多次状态）
  → 第三方（可选）：HMAC GET .../print-requests/{request_id} 轮询
```

`terminal_ticket`：不透明字符串；约 **5 分钟**有效；选定入口后绑定到你们的 `code`；成功创建 print-request 后消耗，不可复用。

## 3. 第三方必须实现的能力

### 3.1 Entry 页

| 项 | 约定 |
|----|------|
| URL | 配置在 Provider `entry_url` |
| 入参 | Query：`terminal_ticket`（Cloud 追加/覆盖） |
| 行为 | 展示业务 UI；**原样保存**该 ticket，提交打印请求时放回 JSON |
| 不要 | 改写 ticket、直连打印机、要求用户再扫一次无关码 |

### 3.2 文件托管（供 Cloud 拉取）

创建 print-request 时提供 `file.url`。Cloud **服务端**下载（非手机下载）。

| 约束 | 说明 |
|------|------|
| scheme | **`http` 或 `https`**（须为 Cloud 默认信任的证书；不支持自签） |
| 主机 | 必须命中 `allowed_file_hosts` |
| 私网 IP | 默认拒绝；仅当 `allow_private_file_hosts=true` 且主机在白名单才允许 RFC1918 |
| 内容 | 与声明的 `size`、`sha256`（64 位小写 hex）、`mime_type` 一致 |
| 时效 | `file.expires_at` 为未来 RFC3339；在过期前保持可下载 |
| 重定向 | Cloud 最多跟随 3 次，每跳重新校验主机 |

### 3.3 状态回调接收（必须）

| 项 | 约定 |
|----|------|
| 方法 / 路径 | `POST {callback_base_url}/api/print/callback` |
| 路径用于验签 | **固定**字符串 `/api/print/callback`（不是带域名的完整 URL） |
| 成功响应 | HTTP **2xx**（建议 `204`）；非 2xx 会触发重试 |
| 幂等 | 按 `event_id` 去重；重复投递应仍返回 2xx |
| 超时 | Cloud 单次约 10s；最多约 **3** 次，间隔约 1m → 2m |

**回调 Headers（出站 HMAC，Cloud → 你们）：**

| Header | 含义 |
|--------|------|
| `X-FP-Client` | 你们的 `code` |
| `X-FP-Timestamp` | Unix 秒；建议校验 ±300s |
| `X-FP-Nonce` | 一次性 |
| `X-FP-Signature` | Base64(HMAC-SHA256)，密钥 = **出站** secret |

**回调 Body：**

```json
{
  "event_id": "uuid",
  "request_id": "uuid",
  "external_order_id": "你们的业务单号",
  "flyprint_job_id": "uuid或null",
  "status": "waiting_terminal",
  "error_code": null,
  "error_message": null,
  "terminal_name": "终端显示名",
  "updated_at": "2026-07-22T06:15:00Z"
}
```

常见 `status`：`waiting_file` → `waiting_terminal` → `dispatched` → `printing` → `completed` / `failed` / `expired` / `cancelled`。

## 4. 第三方调用 FlyPrint 的 API

Base：`{CLOUD_ROOT}/api/v1/integrations/{code}/print-requests`  
`{CLOUD_ROOT}` 由 FlyPrint 提供（经 Nginx 的对外根，例如 `http://print.example.com`）。

### 4.1 HMAC（入站：你们 → Cloud）

**Headers（每个请求必带）：**

| Header | 规则 |
|--------|------|
| `X-FP-Client` | 必须等于路径里的 `{code}` |
| `X-FP-Timestamp` | Unix 秒；Cloud 窗口 ±300s |
| `X-FP-Nonce` | 每个 provider 唯一；300s 内不可复用 |
| `X-FP-Signature` | 见下 |
| `Content-Type` | `application/json`（有 body 时） |

**规范串（UTF-8，`\n` 连接）：**

```text
{METHOD}
{PATH}
{timestamp}
{nonce}
{sha256_hex(raw_body)}
```

- `PATH`：请求 URL **path**（含 `/api/v1/...`），**不含** query。  
  - 创建：`/api/v1/integrations/{code}/print-requests`  
  - 查询：`/api/v1/integrations/{code}/print-requests/{request_id}`  
- GET 无 body 时对 **空 body** 做 SHA256。  
- 签名：`Base64(HMAC-SHA256(入站密钥, 规范串))`（密钥为创建时给出的 Raw Base64 字符串，按实现原样用作 HMAC key）。

参考：`api/internal/integration/hmac.go`；Demo：`integration-demo/main.go` 中 `flyPrintRequest`。

### 4.2 创建打印请求

`POST /api/v1/integrations/{code}/print-requests`

**Body：**

```json
{
  "external_order_id": "必填，你们侧幂等键",
  "external_user_id": "可选",
  "external_user_name": "可选",
  "terminal_ticket": "必填，entry 拿到的原样字符串",
  "file": {
    "url": "http://files.example.com/a.pdf",
    "expires_at": "2026-07-22T07:00:00Z",
    "name": "a.pdf",
    "size": 12345,
    "mime_type": "application/pdf",
    "sha256": "64位小写hex"
  },
  "print_options": {
    "copies": 1,
    "paper_size": "A4",
    "color_mode": "grayscale",
    "duplex_mode": "single"
  },
  "metadata": { "任意": "JSON对象，可选" }
}
```

| 字段 | 要求 |
|------|------|
| `external_order_id` | 必填；同 code 下相同单号 + 相同 body hash → 幂等返回；不同参数 → 409 |
| `terminal_ticket` | 必填；须已为你们 provider 选定且未过期 |
| `file.*` | `url/name/size/sha256/expires_at` 必填；`size>0`；MIME/大小受 Provider 策略约束 |
| `print_options.copies` | 1–99 |

**成功：`202 Accepted`（扁平 JSON，无 `code/message` 包装）：**

```json
{
  "request_id": "uuid",
  "status": "waiting_file",
  "idempotent": false
}
```

| HTTP | 含义（摘要） |
|------|----------------|
| 400 | JSON/字段校验失败 |
| 401 | Client/签名/Provider 不可用 |
| 403 | 源 IP 不在 `allowed_ip_cidrs` |
| 409 | 单号冲突 / ticket 不可用 / nonce 重放 |
| 500 | 内部或 Redis nonce 不可用 |

错误体多为：`{ "code": <http>, "message": "..." }`。

### 4.3 查询打印请求

`GET /api/v1/integrations/{code}/print-requests/{request_id}`  
同样入站 HMAC（body 为空）。

**成功：`200`，管理端风格包装：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "...",
    "provider_code": "...",
    "external_order_id": "...",
    "status": "waiting_terminal",
    "flyprint_job_id": "...",
    "error_code": null,
    "error_message": null,
    "...": "其余非敏感字段"
  }
}
```

不返回：`file_url`、原始 ticket、密钥。

### 4.4 状态机（对接方需认识）

| status | 含义 |
|--------|------|
| `waiting_file` | 已受理，正在/即将拉文件 |
| `waiting_terminal` | 文件就绪，等用户在 Edge 确认 |
| `dispatched` | 已确认，任务已下发 |
| `printing` | 打印中 |
| `completed` | 成功 |
| `failed` | 失败（看 `error_code` / `error_message`） |
| `expired` | 票据/请求过期 |
| `cancelled` | 取消（如节点删除等） |

**终态以回调为准**；轮询为辅。

## 5. FlyPrint 侧 Provider 配置字段（运维对照）

与 §1.1 一一对应。额外开关：

| 字段 | 含义 |
|------|------|
| `enabled` | 是否接受 print-requests / 发回调；创建后默认常为关，配齐策略与密钥后再开 |
| `entry_visible` | 是否在公共 `/entry` 页展示（通常需同时 `enabled`） |

创建成功响应会带一次性：

```json
{
  "data": {
    "provider": { "...": "非密钥字段" },
    "inbound_hmac_secret": "...",
    "outbound_hmac_secret": "..."
  }
}
```

`Cache-Control: no-store`。密钥不会再次从 API 读出，丢失只能轮换。

## 6. 默认可达性假设（按「正式对接」）

| 链路 | 要求 |
|------|------|
| 手机 → Cloud `/entry` | 用户网络可达 Cloud 根（Edge `cloud.base_url` 指向该根；可为 `http://` 或 `https://`） |
| 手机 → `entry_url` | 用户网络可达第三方入口（http 或 https） |
| 第三方服务器 → Cloud `/api/v1/integrations/...` | 出口 IP 落在 `allowed_ip_cidrs` |
| Cloud → `callback_base_url` | Cloud 出站可达（http 或 https，受信证书） |
| Cloud → `file.url` | Cloud 出站可达；主机在白名单；http 或 https |

协议与 HMAC **与部署环境无关**；变的是 URL 与 CIDR/主机名。http 与 https **双兼容**，不强制正式环境只能 https（强制策略另议）。

Edge / 打印机 / 官方扫码上传 **不属于**第三方对接范围（见 [`M0演示交接手册.md`](M0演示交接手册.md)）。Edge↔Cloud 传输另支持 `http↔ws`、`https↔wss`。

## 7. 局域网与 Compose Demo 特例

仅当 Cloud、第三方、手机在同一局域网，或使用仓库自带 Demo 时查阅。

| 场景 | 怎么改配置（协议不变） |
|------|------------------------|
| 教室 Wi‑Fi 演示 | `entry_url` / Cloud 根用局域网 IP；`allowed_ip_cidrs` 覆盖对方出口；文件主机可为内网名并开 `allow_private_file_hosts` |
| Compose **同机**自带 Demo | `code=livacloud-demo`；`entry_url=http://<LAN-IP>:8012/integration-demo/entry`；`callback_base_url=http://integration-demo:8080`；`allowed_file_hosts=integration-demo`；CIDR 常用 `172.16.0.0/12`（覆盖 Docker 桥，**勿写 127.x**） |
| 不想跑 Demo | 不创建该 Provider 即可；对接自有系统时 **不要**照抄容器名 |
| Edge 扫码 | `cloud.base_url` 填手机可达的 Cloud 根（局域网可用 `http://<IP>:8012`；公网用 `https://域名`）。**勿**给手机用 `localhost`；**勿**用 `https://localhost` 再指望 Edge 改写成局域网 IP（证书主机名会失败） |

Demo setup 页（粘贴 HMAC）：`/integration-demo/setup`。完整局域网运维步骤见交接手册，**不是**第三方开发主路径。

## 8. 最小联调检查单

第三方：

- [ ] Entry 能收下并回传 `terminal_ticket`  
- [ ] 入站 HMAC 创建请求得 `202` + `request_id`  
- [ ] 实现 `POST /api/print/callback` 验出站签 + `event_id` 幂等  
- [ ] 文件 URL 在过期前可被 Cloud 拉取且 hash 一致  

FlyPrint：

- [ ] Provider 字段与 §1.1 一致并已 `enabled`  
- [ ] 入站/出站密钥已交给对方（或已轮换对齐）  
- [ ] Redis 可用；Edge 在线且默认可打印（用于端到端一单）
