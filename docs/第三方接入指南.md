# 飞印（FlyPrint）第三方接入指南

| 属性 | 说明 |
|------|------|
| 文档类型 | 集成接口契约与联调说明 |
| 适用对象 | 第三方业务系统开发方；飞印侧运维在管理端配置 Provider |
| 默认场景 | 双方网络互相可达的正式对接（公网或固定专线） |
| 相关文档 | [系统说明](系统说明.md) · [部署与验证](部署与验证.md) |
| 修订说明 | 本文为接口与安全约定的权威说明；局域网与自带演示见第 8 节 |

本文约定第三方业务系统与飞印 Cloud 的对接方式。产品流程与安全边界见 [系统说明](系统说明.md)；Cloud / Edge 安装与验证见 [部署与验证](部署与验证.md)。

---

## 1. 协作模型

推荐协作顺序：

1. 第三方按本文完成入口页、文件托管、HMAC 调用与回调接收  
2. 双方交换第 2 节所列必要信息及 HMAC 密钥  
3. 飞印管理端创建并启用集成 Provider  
4. 完成一单端到端联调后，日常运行无需再约定额外协议  

### 1.1 职责边界

| 方 | 职责 | 不得实施 |
|----|------|----------|
| 第三方 | 业务入口、选文件/下单、调用 Cloud 集成 API、接收状态回调 | 直连 Edge 或现场打印机；跳过一体机确认 |
| 飞印 Cloud | 校验票据与签名、拉取文件、向 Edge 下发预览与打印、回传状态 | — |
| 用户（一体机） | 在 Edge 预览并确认打印参数 | — |

用户必须在 **Edge 一体机**上预览并确认后，系统才会创建正式打印任务。

### 1.2 传输与 URL 约定

| 约定 | 说明 |
|------|------|
| 协议 | Provider 的 `entry_url`、`callback_base_url` 以及请求中的 `file.url` 支持 `http` 与 `https`；URL 不得包含 userinfo。正式对接推荐使用 `https` |
| 证书 | **不支持自签证书**作为默认可信：Cloud 出站拉取文件与发送回调使用系统默认信任链 |
| Redis | 启用 Provider 前，Cloud 侧须已配置 Redis（用于 nonce 防重放） |

协议与 HMAC 算法与部署环境无关；随环境变化的是可达 URL、出口 IP/CIDR 与文件主机白名单。

## 2. 一次交换清单

对接完成时，双方仅交换下列信息，不另行发明口令或会话流程。

### 2.1 第三方提供给飞印（用于创建 Provider）

| 字段 | 说明 | 示例形态 |
|------|------|----------|
| `code` | 稳定唯一标识；用于路径与 `X-FP-Client`；须匹配 `^[a-z][a-z0-9-]{1,62}$` | `acme-hr` |
| `display_name` | 公共入口 `/entry` 选择页上的按钮文案 | `ACME 人事打印` |
| `entry_url` | 用户手机浏览器打开的业务入口（Cloud 会追加 `terminal_ticket`） | `https://third.example.com/flyprint/entry` |
| `callback_base_url` | Cloud 出站可达的回调基址；实际请求为 `{base}/api/print/callback` | `https://third.example.com` |
| `allowed_ip_cidrs` | 第三方调用 Cloud 集成 API 时的出口 IP/CIDR（CSV） | `203.0.113.10/32` |
| `allowed_file_hosts` | `file.url` 允许的主机名（CSV，不含 scheme 与路径） | `files.third.example.com` |
| `allow_private_file_hosts` | 文件主机解析到 RFC1918 时是否允许（仍须在白名单内） | 公网对接通常为 `false` |
| `max_file_size` | 单文件字节上限 | `10485760` |
| `allowed_mime_types` | MIME 白名单（CSV） | `application/pdf` |

### 2.2 飞印提供给第三方

| 信息 | 说明 |
|------|------|
| Cloud 集成 API 根地址 | 对外服务根，例如 `https://print.example.com`；下列路径挂载于 `/api/v1/...` |
| `code` | 与双方约定一致 |
| 入站 HMAC 密钥 `inbound_hmac_secret` | 第三方签署「调用 Cloud」的请求；创建或轮换时**仅展示一次** |
| 出站 HMAC 密钥 `outbound_hmac_secret` | 第三方验证「Cloud 回调」的签名；同上，仅展示一次 |

### 2.3 HMAC 密钥配对

| 方向 | 密钥 | 签署方 | 验证方 |
|------|------|--------|--------|
| 第三方 → 飞印 | 入站（inbound） | 第三方 | Cloud |
| 飞印 → 第三方 | 出站（outbound） | Cloud | 第三方 |

密钥轮换：在飞印管理端执行「轮换密钥」后，双方须同步更新；旧密钥立即失效。

## 3. 端到端流程

```text
用户在 Edge 扫码
  → 手机打开 Cloud GET /entry?...
  → Cloud 签发 terminal_ticket（约 5 分钟有效）
  → 用户在 /entry 选择第三方 display_name
  → Cloud 内部：POST /api/v1/public/terminal-entry/select（手机浏览器调用，第三方不实现）
  → Cloud 302 引导至：
       GET {entry_url}?terminal_ticket=<opaque>
  → 第三方：完成登录 / 选文件 / 确认业务单
  → 第三方：HMAC POST /api/v1/integrations/{code}/print-requests
       （请求体携带同一 terminal_ticket 与文件元数据）
  → Cloud：拉取 file.url → 向 Edge 下发预览
  → 用户在 Edge 确认参数
  → Cloud：创建打印任务 → Edge 经 IPP 出纸
  → Cloud：HMAC POST {callback_base_url}/api/print/callback（多次状态）
  → 第三方（可选）：HMAC GET .../print-requests/{request_id} 查询状态
```

`terminal_ticket` 为不透明字符串，约 **5 分钟**有效；用户在 `/entry` 选定入口后绑定至对应 Provider 的 `code`。

> **票据一次性（重要）**  
> 成功创建 print-request（HTTP `202`）后，该 `terminal_ticket` **立即被消耗**，不可再次用于创建请求。  
> - 若收到 `409` 且 `message` 为 `terminal ticket is unavailable for this provider`：**须让用户返回终端重新扫码**获取新票据。  
> - 若在**尚未成功创建**（如 `400` / `401` / `403`，或网络超时未收到 `202`）时失败，且票据仍处于有效选中状态，可用**同一 ticket**、**新的 nonce** 重试；幂等重试见第 5.2 节。  
> - 已成功创建后，后续状态变更请用返回的 `request_id` 接收回调或查询，**不要**复用同一 ticket 再下单。

Cloud 在用户选入口时调用的 `POST /api/v1/public/terminal-entry/select` 由 `/entry` 页面发起，**不属于**第三方须实现的 API；排障时可对照该步骤是否成功（例如入口未启用、票据过期等）。

## 4. 第三方须实现的能力

### 4.1 业务入口页（Entry）

| 项 | 约定 |
|----|------|
| URL | 配置为 Provider 的 `entry_url` |
| 入参 | Query 参数 `terminal_ticket`（由 Cloud 追加或覆盖） |
| 行为 | 展示业务界面；**原样保存**该票据，并在创建打印请求时原样回传 |
| 禁止 | 改写票据、直连打印机、要求用户再次扫描与本流程无关的二维码 |

### 4.2 文件托管（供 Cloud 拉取）

创建 print-request 时须提供 `file.url`。文件由 **Cloud 服务端**下载，而非用户手机下载。

| 约束 | 说明 |
|------|------|
| 协议 | `http` 或 `https`（须为 Cloud 默认可信证书；不支持自签） |
| 主机 | 必须命中 `allowed_file_hosts` |
| 私网地址 | 默认拒绝；仅当 `allow_private_file_hosts=true` 且主机在白名单内时允许 RFC1918 |
| 内容一致性 | 与声明的 `size`、`sha256`（64 位小写十六进制）、`mime_type` 一致 |
| `mime_type` | **必填**；须命中 Provider 的 `allowed_mime_types`；Cloud 会校验文件头与声明一致 |
| 时效 | `file.expires_at` 为未来时刻的 RFC3339；在过期前保持可下载 |
| 重定向 | Cloud 最多跟随 3 次，每一跳重新校验主机是否允许 |
| 下载方式 | Cloud 使用 `GET` 拉取；**不要求**事先 `HEAD`；以实际下载字节数与 `size` 比对 |
| URL 鉴权 | 允许 query 参数（如 `?token=...`），但主机名仍须在 `allowed_file_hosts` 内 |

**示例（演示环境，容器内可达）：**

```text
http://integration-demo:8080/files/{file_id}?expires={unix}
```

公网对接时须使用 Cloud 出站可达的 HTTPS 主机，且该主机已写入白名单。

### 4.3 状态回调接收（必须）

| 项 | 约定 |
|----|------|
| 方法与路径 | `POST {callback_base_url}/api/print/callback` |
| 验签用路径 | 固定字符串 `/api/print/callback`（不含域名） |
| 成功响应 | HTTP **2xx**（建议 `204`）；非 2xx 将触发重试 |
| 幂等 | 按 **`event_id`** 去重；同一 `event_id` 重复投递仍应返回 2xx |
| 出站 nonce | 每次回调携带新的 `X-FP-Nonce`；建议第三方在 ±300 秒窗口内**拒收重复 nonce**（辅助防重放，与 `event_id` 幂等并存） |
| 超时与重试 | 单次 HTTP 超时 **10 秒**；失败时最多再投递 **2** 次（合计 **3** 次尝试）；第 2、3 次间隔分别为 **1 分钟、2 分钟**（指数退避，上限 3 小时） |

**回调请求头（出站 HMAC，Cloud → 第三方）：**

| Header | 含义 |
|--------|------|
| `X-FP-Client` | Provider 的 `code` |
| `X-FP-Timestamp` | Unix 秒；建议校验 ±300 秒 |
| `X-FP-Nonce` | 一次性标识 |
| `X-FP-Signature` | Base64(HMAC-SHA256)，密钥为**出站**密钥 |

**回调请求体字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `event_id` | string | 本次回调唯一 ID；**幂等键** |
| `request_id` | string | 集成 print-request UUID |
| `external_order_id` | string | 创建请求时的业务单号 |
| `flyprint_job_id` | string \| null | 关联 Cloud 打印任务 ID；早期状态可能为 `null` |
| `status` | string | 当前集成状态（见 §5.4） |
| `error_code` | string \| null | 失败或取消时的稳定错误码；成功态常为 `null` |
| `error_message` | string \| null | 人类可读说明 |
| `terminal_name` | string | 目标终端/打印机展示名（可能为空字符串） |
| `updated_at` | string | RFC3339 UTC 时间戳 |

**回调请求体示例：**

```json
{
  "event_id": "uuid",
  "request_id": "uuid",
  "external_order_id": "第三方业务单号",
  "flyprint_job_id": "uuid或null",
  "status": "waiting_terminal",
  "error_code": null,
  "error_message": null,
  "terminal_name": "终端显示名",
  "updated_at": "2026-07-22T06:15:00Z"
}
```

常见状态迁移：`waiting_file` → `waiting_terminal` → `dispatched` → `printing` → `completed` / `failed` / `expired` / `cancelled`。

## 5. 第三方调用飞印的 API

基址：`{CLOUD_ROOT}/api/v1/integrations/{code}/print-requests`  
其中 `{CLOUD_ROOT}` 由飞印提供（经反代后的对外根地址）。

> **响应格式不一致（易踩坑）**  
> - **创建** `POST .../print-requests`：成功为 **`202`**，响应体为**扁平 JSON**（`request_id`、`status`、`idempotent`），**无** `{code, message, data}` 包装。  
> - **查询** `GET .../print-requests/{id}`：成功为 **`200`**，响应体为 **`{ "code": 200, "message": "success", "data": { ... } }`** 包装。  
> Demo 与客户端解析逻辑须按方法区分，不可共用同一反序列化结构。

### 5.1 入站 HMAC（第三方 → Cloud）

**每个请求须携带的请求头：**

| Header | 规则 |
|--------|------|
| `X-FP-Client` | 必须等于路径中的 `{code}` |
| `X-FP-Timestamp` | Unix 秒；Cloud 校验窗口为 ±300 秒 |
| `X-FP-Nonce` | 在同一 Provider 下唯一；300 秒内不可复用 |
| `X-FP-Signature` | 见下方规范串 |
| `Content-Type` | 有请求体时为 `application/json` |

**规范串（UTF-8，以 `\n` 连接）：**

```text
{METHOD}
{PATH}
{timestamp}
{nonce}
{sha256_hex(raw_body)}
```

| 规则 | 说明 |
|------|------|
| `PATH` | 请求 URL 的 path（含 `/api/v1/...`），**不含** query |
| 创建 | `/api/v1/integrations/{code}/print-requests` |
| 查询 | `/api/v1/integrations/{code}/print-requests/{request_id}` |
| 空体 | GET 等无 body 时，对空 body 计算 SHA-256 |
| 签名 | `Base64(HMAC-SHA256(入站密钥, 规范串))`；密钥为创建时给出的字符串，按实现原样用作 HMAC key |

参考实现见仓库中的集成演示程序（`integration-demo/`）及 Cloud 侧 HMAC 模块。

### 5.2 创建打印请求

`POST /api/v1/integrations/{code}/print-requests`

**请求体：**

```json
{
  "external_order_id": "必填，第三方侧幂等键",
  "external_user_id": "可选",
  "external_user_name": "可选",
  "terminal_ticket": "必填，入口页取得的原样字符串",
  "file": {
    "url": "https://files.example.com/a.pdf",
    "expires_at": "2026-07-22T07:00:00Z",
    "name": "a.pdf",
    "size": 12345,
    "mime_type": "application/pdf",
    "sha256": "64位小写hex"
  },
  "print_options": {
    "copies": 1,
    "paper_size": "A4",
    "color_mode": "grayscale",
    "duplex_mode": "single"
  },
  "metadata": { "任意": "JSON对象，可选" }
}
```

| 字段 | 要求 |
|------|------|
| `external_order_id` | 必填；同一 `code` 下相同单号且**原始请求体字节级 SHA-256 相同** → 幂等返回；参数不同 → `409` |
| `terminal_ticket` | 必填；须已在 `/entry` 为本 Provider 选定且未过期；**成功 `202` 后不可复用**（见 §3） |
| `file.url` / `file.name` / `file.size` / `file.sha256` / `file.expires_at` / `file.mime_type` | **均必填**；`size > 0`；`mime_type` 须命中 Provider 的 `allowed_mime_types` |
| `print_options` | **至少含 `copies`（1–99）**；省略整个对象时 `copies` 默认为 0 → **`400`**；其余字段见 §5.2.1 |

**幂等与重试：** 系统对**原始 HTTP body 字节**计算 SHA-256 作为 `request_hash`。JSON 语义相同但字段顺序、空格不同 → 视为不同请求；同一 `external_order_id` 会返回 **`409 external_order_id already exists with different parameters`**。网络重试须发送**字节级相同**的 body，并更换 `X-FP-Nonce`。

#### 5.2.1 `print_options` 约定值

创建请求时提交的打印参数为**建议值**：用户仍可在 Edge 预览页修改后确认。Cloud 入站校验强制约束 `copies`；省略 `print_options` 或省略 `copies` 均会导致 **`400`**（`print_options.copies must be between 1 and 99`）。

| 字段 | 推荐取值 | 说明 |
|------|----------|------|
| `copies` | `1`–`99` | 必填语义；超出范围返回 `400` |
| `paper_size` | 如 `A4` | 字符串；最终须为打印机支持的介质名 |
| `color_mode` | `grayscale` · `color` | 黑白 / 彩色。终端侧亦接受 `mono`、`monochrome` 等别名并归一化 |
| `duplex_mode` | `single` · `duplex` | 单面 / 双面。终端确认时可能映射为长边双面等设备能力 |

联调示例可使用：`{"copies":1,"paper_size":"A4","color_mode":"grayscale","duplex_mode":"single"}`。

**成功响应：`202 Accepted`（扁平 JSON，无统一 `code`/`message` 包装）：**

```json
{
  "request_id": "uuid",
  "status": "waiting_file",
  "idempotent": false
}
```

| HTTP | 含义（摘要） |
|------|----------------|
| 400 | JSON 或字段校验失败 |
| 401 | Client、签名或 Provider 不可用 |
| 403 | 源 IP 不在 `allowed_ip_cidrs` |
| 409 | 单号冲突、票据不可用或 nonce 重放 |
| 500 | 内部错误或 Redis nonce 不可用 |

错误响应体多为：`{ "code": <http>, "message": "..." }`。

### 5.3 查询打印请求

`GET /api/v1/integrations/{code}/print-requests/{request_id}`  
同样使用入站 HMAC（请求体为空）。

**成功响应：`200`（管理端风格包装）：**

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "id": "...",
    "provider_code": "...",
    "external_order_id": "...",
    "status": "waiting_terminal",
    "flyprint_job_id": "...",
    "error_code": null,
    "error_message": null
  }
}
```

响应中不包含：`file_url`、原始票据、密钥等敏感字段。

**不存在时：`404 Not Found`**

```json
{
  "code": 404,
  "message": "integration print request not found"
}
```

### 5.4 状态说明

| status | 含义 |
|--------|------|
| `waiting_file` | 已受理，正在或即将拉取文件 |
| `waiting_terminal` | 文件就绪，等待用户在 Edge 确认 |
| `dispatched` | 已确认，任务已下发 |
| `printing` | 打印中 |
| `completed` | 成功 |
| `failed` | 失败（参见 `error_code` / `error_message`） |
| `expired` | 票据或请求过期 |
| `cancelled` | 已取消（例如关联节点删除等） |

**终态以回调为准**；查询接口可作为辅助手段。

### 5.5 `error_code` 对照（摘要）

回调与查询响应中，非成功终态可能携带 `error_code` 与 `error_message`。对接方应以 **`status` + `error_code`** 做业务分支；未知码按失败兜底，并保留 `error_message` 便于排障。

**集成链路（Cloud 侧常见）：**

| error_code | 典型 status | 含义 |
|------------|-------------|------|
| `provider_unavailable` | `failed` | Provider 不可用或已停用，文件阶段失败 |
| `file_validation_failed` | `failed` | 文件拉取/校验失败（可达性、大小、哈希、MIME、过期等） |
| `file_record_failed` | `failed` | 文件已校验但 Cloud 落库失败 |
| `terminal_ticket_expired` | `expired` | 终端票据或请求在等待确认前过期 |
| `node_deleted` | `cancelled` | 目标边缘节点被删除，关联请求取消 |

**打印链路（Edge / 设备侧常见，随作业终态回传）：**

| error_code | 含义（摘要） |
|------------|----------------|
| `printer_out_of_paper` | 缺纸 |
| `printer_out_of_toner` | 缺粉 / 耗材耗尽 |
| `printer_jammed` | 卡纸 |
| `printer_cover_open` | 盖板未关闭 |
| `printer_offline` | 打印机离线 |
| `printer_busy` | 打印机忙 |
| `printer_user_intervention` | 需人工干预 |
| `ipp_unreachable` / `ipp_submission_failed` | IPP 不可达或提交失败 |
| `ipp_parameter_unsupported` | 当前参数不被设备支持 |
| `document_unsupported` / `document_conversion_failed` | 文档类型或转换失败 |
| `print_canceled` | 打印被取消 |
| `print_timeout` | 打印超时 |
| `result_unconfirmed` | 结果未确认（对应作业 `unconfirmed`；集成侧常映射为失败类终态，**禁止静默重打**） |
| `dispatch_ack_timeout` | 下发确认超时（Cloud 侧未确认类场景） |

上表为常见码摘要，并非封闭枚举；设备与版本可能产生其他稳定字符串码。新码出现时仍应依据 `status` 判断是否终态。

### 5.6 HTTP 错误与典型 `message`

集成 API 错误体多为 `{ "code": <http>, "message": "<英文原文>" }`（创建成功 `202` 除外）。常见如下：

| HTTP | 典型 `message` | 含义 |
|------|----------------|------|
| 400 | `invalid integration request JSON` | 请求体非合法 JSON |
| 400 | `external_order_id, terminal_ticket, and complete file metadata are required` | 必填字段缺失 |
| 400 | `file.expires_at must be RFC3339` / `file.expires_at must be in the future` | 文件过期时间无效 |
| 400 | `file violates provider size or MIME policy` | 大小超限、`mime_type` 为空或未在白名单 |
| 400 | `print_options.copies must be between 1 and 99` | 未传 `print_options.copies` 或超出范围 |
| 401 | `provider identity does not match request path` | `X-FP-Client` 与路径 `{code}` 不一致 |
| 401 | `provider is unavailable` | Provider 不存在或未启用 |
| 401 | `invalid integration signature` | HMAC 或时间戳/nonce 校验失败 |
| 403 | `source IP is not allowed` | 源 IP 不在 `allowed_ip_cidrs`（见 §7.1） |
| 409 | `external_order_id already exists with different parameters` | 同单号但 body 字节摘要不同 |
| 409 | `terminal ticket is unavailable for this provider` | 票据已消耗、未选中、过期或 Provider 不匹配 → **须重新扫码** |
| 409 | `integration nonce was already used` | 300 秒内 nonce 重复；更换 nonce 后可重试 |
| 500 | `integration nonce store is unavailable` | Redis 不可用 |
| 404 | `integration print request not found` | 查询的 `request_id` 不存在（包装在 `{code,message}` 中） |

## 6. 飞印侧 Provider 配置

字段与第 2.1 节一一对应。额外开关：

| 字段 | 含义 |
|------|------|
| `enabled` | 是否接受 print-requests 并发送回调；创建后默认常为关闭，策略与密钥配齐后再启用 |
| `entry_visible` | 是否在公共 `/entry` 页展示（通常需同时 `enabled`） |

**配置方式：** 日常由 Cloud **管理端 UI** 完成。运维自动化可使用管理端 REST（须 OAuth2 管理员令牌，路径前缀 `/api/v1/admin/integration-providers`：创建、更新、启停、轮换密钥等）；**第三方业务系统不调用**这些接口。

创建成功时，响应中会一次性返回：

```json
{
  "data": {
    "provider": { "...": "非密钥字段" },
    "inbound_hmac_secret": "...",
    "outbound_hmac_secret": "..."
  }
}
```

响应带 `Cache-Control: no-store`。密钥不会再次通过 API 读出；丢失后只能轮换。

## 7. 可达性要求（正式对接）

| 链路 | 要求 |
|------|------|
| 手机 → Cloud `/entry` | 用户网络可达 Cloud 根；Edge 的 `cloud.base_url` 指向该根（可为 `http://` 或 `https://`） |
| 手机 → `entry_url` | 用户网络可达第三方入口 |
| 第三方服务器 → Cloud 集成 API | 出口 IP 落在 `allowed_ip_cidrs` |
| Cloud → `callback_base_url` | Cloud 出站可达（`http` 或 `https`，受信证书） |
| Cloud → `file.url` | Cloud 出站可达；主机在白名单内 |

### 7.1 源 IP 校验与反向代理

集成 API 使用 Gin `ClientIP()` 校验 `allowed_ip_cidrs`。Cloud 经 Nginx 或外层反代时，须同时配置：

| 项 | 说明 |
|----|------|
| Provider `allowed_ip_cidrs` | 填写**第三方服务器真实出口 IP/CIDR**（不是 Cloud 内网地址） |
| Cloud `trusted_proxy_cidrs` | 填写**紧邻 API 的可信反代**地址段，使 API 能从 `X-Forwarded-For` 解析真实客户端 IP |

Compose / 公网示例：

- 环境变量 `TRUSTED_PROXY_CIDRS`（映射为 `FLY_PRINT_SERVER_TRUSTED_PROXY_CIDRS`），例如 Docker 网桥 `172.16.0.0/12`，或外层 TLS 反代所在网段。  
- 未配置可信代理时，API 可能只看到 Nginx 容器 IP → 第三方真实 IP 无法匹配 → **`403 source IP is not allowed`**。  
- 同机集成演示中，演示程序直连 `http://api:8080`，其源 IP 为 Docker 网段，故演示 Provider 常用 `172.16.0.0/12`；**勿**写 `127.0.0.0/8`。

部署侧变量说明见 [部署与验证](部署与验证.md) §2.2。网络地址三类可达性见 [系统说明](系统说明.md) §2.3。

### 7.2 回调与入口地址勿混用

| 地址 | 谁访问 | 示例 |
|------|--------|------|
| `entry_url` | **用户手机** | `https://third.example.com/flyprint/entry` 或 `http://<LAN-IP>:8012/integration-demo/entry` |
| `callback_base_url` | **Cloud 出站** | `https://third.example.com` 或 Compose 内 `http://integration-demo:8080` |
| `file.url` 主机 | **Cloud 出站** | 须列入 `allowed_file_hosts`；可与 `callback_base_url` 同域或不同域 |

手机可达地址与 Cloud 容器可达地址**通常不同**；Compose 同机联调时回调/文件用**服务名**，入口用手机 LAN IP + 反代前缀。

Edge、现场打印机以及飞印官方扫码上传**不属于**第三方对接范围，见 [系统说明](系统说明.md) 与 [部署与验证](部署与验证.md)。Edge 与 Cloud 之间的控制通道另支持 `http`↔`ws` 与 `https`↔`wss`。

## 8. 局域网与集成演示

仅当 Cloud、第三方与手机处于同一局域网，或使用产品自带的集成演示程序时，按本节调整**配置值**；协议与 HMAC 不变。

| 场景 | 配置要点 |
|------|----------|
| 局域网演示 | `entry_url` 与 Cloud 根使用局域网可达地址；`allowed_ip_cidrs` 覆盖对方出口；文件主机可为内网名，并视需要开启 `allow_private_file_hosts` |
| Compose 同机集成演示 | 见下表；Provider **`code` 固定为 `livacloud-demo`**（与示例 `acme-hr` 不同，勿抄错） |
| 不启用演示 | 不创建对应 Provider 即可；对接自有系统时勿照搬演示主机名 |
| Edge 扫码地址 | `cloud.base_url` 须为手机可达的 Cloud 根。勿对手机链路使用 `localhost`；勿依赖「`https://localhost` 再改写为局域网 IP」（证书主机名将不匹配） |

**Compose 同机演示 Provider 参考值：**

| 字段 | 值 |
|------|-----|
| `code` | `livacloud-demo` |
| `display_name` | `丽娃云聘 Demo`（可自定） |
| `entry_url` | `http://<Cloud局域网IP>:8012/integration-demo/entry` |
| `callback_base_url` | `http://integration-demo:8080`（Cloud 容器内可达，**非**手机地址） |
| `allowed_ip_cidrs` | `172.16.0.0/12`（覆盖 Docker 桥接网段） |
| `allowed_file_hosts` | `integration-demo` |
| `allow_private_file_hosts` | `true` |

演示程序**容器内**路由为 `/setup`；经 Nginx 反代后对外为 **`/integration-demo/setup`**（以实际反代前缀为准）。同机联调步骤见 [部署与验证](部署与验证.md) §7.3；该路径**不是**第三方开发的主文档路径。

## 9. 联调验收要点

**第三方侧：**

1. 入口页能够接收并原样回传 `terminal_ticket`  
2. 入站 HMAC 创建请求返回 `202` 与 `request_id`  
3. 实现 `POST /api/print/callback`，校验出站签名，并按 `event_id` 幂等处理  
4. 文件 URL 在过期前可被 Cloud 拉取，且内容与声明的摘要一致  

**飞印侧：**

1. Provider 字段与第 2.1 节一致，且已启用  
2. 入站/出站密钥已安全交付对方（或轮换后已对齐）  
3. Redis 可用；用于端到端验证的 Edge 在线且已配置默认可打印设备
