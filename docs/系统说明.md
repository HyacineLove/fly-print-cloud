## 0. 协作模型（读完应能回答）

```text
FlyPrint 是什么？
  → Cloud 控制面 + Edge 一体机；Cloud 不直连打印机
用户怎么打一单？
  → 扫码进门 → 官方上传或第三方下单 → Edge 预览确认 → IPP 出纸
三条地址别混：手机→Cloud、Edge→Cloud、Cloud→第三方回调
安全边界：终端票据 / 一机占用 / 禁止第三方直打 / Edge 默认回环管理
```

**边界（不可违反）：**

- Cloud **不**直连现场打印机；出纸只经 Edge **IPP**。  
- 用户必须在 **Edge 一体机**上预览并确认后，才会真正建打印任务。  
- 第三方 **不**直连 Edge / 打印机；只调 Cloud 集成 API 并收回调。  
- Edge **禁止** Spooler / WSD / RAW 等回退打印链路。

## 1. 是什么

| 仓 | 角色 |
|----|------|
| `fly-print-cloud` | API、Admin、Nginx、Compose、可选 `integration-demo`、文档 |
| `fly-print-edge` | 一体机运行时、用户/管理 UI、启动器、安装包源码 |

**Cloud：** 认证、节点/打印机登记、文件与凭证、打印任务编排、经 WebSocket 向 Edge 下发预览/打印指令。  
**Edge：** 出码、kiosk 用户页、预览确认、文档转 PDF、IPP 提交与终态回传。

技术栈摘要：Cloud = Go/Gin/PG/WS + React Admin；Edge = Python/FastAPI + 静态用户页 + Edge 浏览器 kiosk。Compose 默认统一入口端口 **8012**。

## 2. 端到端主流程

### 2.1 官方打印

```text
Edge 出示二维码（/api/qr_code）
  → 手机打开 Cloud GET /entry?token=...
  → 校验上传凭证后签发 terminal_ticket（固定约 5 分钟，且不超过上传凭证剩余有效期）
  → Cloud 下行 terminal_occupied（Edge ACK；登录页遮挡二维码）
  → 用户选「官方打印」→ Cloud 返回 redirect_url → 手机跳转 /upload 上传文件
  → Cloud 向 Edge 下发 preview_file（带 session + ticket hash）
  → 用户在 Edge 调参数并确认
  → Edge submit_print_params → Cloud 建 print_job
  → Edge 经 IPP 打印 → job_update 回传终态（completed/failed/...）
```

（扫码凭证进 Entry 时，Cloud 会用 HTTP 302 换发带 `terminal_ticket` 的 `/entry`；入口按钮跳转则是接口返回 `redirect_url` 后由页面 `location` 跳转。）

### 2.2 第三方打印

```text
扫码 → /entry → 选第三方 display_name
  → Cloud 返回 redirect_url，手机 location 跳到对方 entry_url?terminal_ticket=...
  → 对方 HMAC POST /api/v1/integrations/.../print-requests
  → Cloud 拉文件 → preview_file → Edge 确认 → 同官方建任务与 IPP
  → 可选 callback 通知第三方
```

契约细节只在 [`第三方接入指南.md`](第三方接入指南.md)；本文不重复 HMAC 字段表。

### 2.3 三条「地址」不要混

| 谁访问谁 | 填什么 | 常见误区 |
|----------|--------|----------|
| 手机 / 浏览器 → Cloud | 用户可达的 Cloud 根（公网多为 `https://域名`） | 给手机写 `localhost` |
| Edge → Cloud | 同上，配在 `cloud.base_url` | 写成仅一体机本机可达的地址 |
| Cloud → 第三方回调 | Cloud **出站**可达的 `callback_base_url` | 把手机用的地址当成容器/内网回调地址 |

Compose 同机 Demo 时，回调常用 `http://integration-demo:8080`（**Cloud 容器 → Demo 容器**），不是手机地址。

## 3. 组件与能力一览

### 3.1 Cloud 管理端（Admin）

| 能力 | 说明 |
|------|------|
| 仪表盘 | 告警与任务趋势；加载失败时为空态 + 错误提示（不注入假数据） |
| 边缘节点 | 激活码、在线状态、别名、启停 |
| 打印机 | 显示名（别名）、启停、所属节点；任务列表无别名时回退设备本名 |
| 打印任务 | 状态、发起方、错误信息 |
| 业务配置 | 上传大小/页数、凭证 TTL、扩展名白名单 |
| 三方接入 | Provider CRUD、HMAC 创建/轮换（只显示一次）、启停 |

登录页与内页同属 Admin；常见 OAuth/校验错误文案经 `admin/src/utils/mapApiError.ts` 映射为中文。  
公开上传页 `/upload` 在扫码官方流程中使用，**不需要**管理端登录（`App.tsx` 独立路由）。

> 源码中另有 `Users` / `Settings` / `OAuth2Clients` 等组件或 Handler，但当前 **未挂菜单与路由**，不算交付能力。

### 3.2 Edge 用户页（kiosk）

| 能力 | 说明 |
|------|------|
| 登录/出码 | 60s 倒计时换码；故障/占用时遮挡二维码 |
| 一机占用 | 进门后遮挡；刷新可重新拉码（作废旧票）；占用态不自动 60s 换码 |
| 预览 | 份数/双面/彩印等；「返回」回扫码页；「立刻打印」提交 |
| 打印中 | 禁止中断回扫码 |
| 完成页 | 成功/失败/缺纸缺粉等提示 |

启动器以 Microsoft Edge **全屏 kiosk** 打开用户页（`--kiosk`、`--edge-kiosk-type=fullscreen`）。

### 3.3 Edge 本机管理

- 配置 `cloud.base_url`、激活、IPP 发现与默认打印机、日志等。  
- 默认监听 **`127.0.0.1:7860`**；管理 API 鉴权当前禁用 → **没有**「Edge 本机管理员密码」。  
- 托盘可「打开管理页」（非 kiosk 窗口）。  
- 改为非回环监听前须补鉴权（仓内硬约束）。

### 3.4 打印与任务状态（程序员需认识）

| 概念 | 要点 |
|------|------|
| 寻址 | 只用 Cloud `printer_id`；显示名不作路由 |
| ACK | Edge 收到并持久化收件 ≠ 打印完成 |
| 终态 | `completed` / `failed` / `canceled` / `unconfirmed`；带稳定 `event_id`，幂等 |
| UNCONFIRMED | 结果不明（如重启）；**禁止**静默重打 |
| 唯一打印链路 | 文档 → 标准 PDF → IPP Print-Job → 设备 job-state=completed |

## 4. 会话、票据与占用

| 对象 | 作用 |
|------|------|
| 上传凭证（QR token） | 扫码进 `/entry`；成功后换独立 `terminal_ticket`；TTL 由业务配置控制（默认约 **180 秒**） |
| `terminal_ticket` | 绑定「这一单 ↔ 这台机/打印机」；固定约 **5 分钟**，且不超过上传凭证剩余有效期；第三方只拿它 |
| `terminal_session_id` + `ticket_hash` | 预览/上传/占用须匹配；刷新二维码会作废未完成票 |
| `terminal_occupied` | Cloud→Edge：遮挡二维码；须 ACK；断线 pending，靠 `terminal_session_state` 补发（已绑定同 hash 时不再重放刷屏） |
| Entry 重选 | ticket 已选（`selected`）但未消费时可退回重选入口；已上传/已提交三方后不可再选 |

## 5. 传输与 URL 约定

- Cloud 对外根、Edge `cloud.base_url`、第三方 `entry_url` / `callback_base_url` / `file.url`：**http 与 https 双兼容**（禁止 URL userinfo）。  
- **不支持自签证书**作出站信任（拉文件 / 回调走系统默认信任链）。正式公网用受信证书。  
- Edge↔Cloud：REST 跟随 `cloud.base_url`；WS 由 `url_scheme.http_url_to_websocket_url` 映射为 **ws / wss**。  
- 出码拼接时若 `cloud.base_url` 含 `localhost`/`127.0.0.1`，Edge **会无条件**改写为本机局域网 IP（**不区分** http/https，见 `main.py` QR 路径）。因此 HTTPS 应直接配置证书域名，**禁止**依赖 `https://localhost` + 改写。

## 6. 明确不做 / 已知缺口

| 项 | 说明 |
|----|------|
| Cloud 直连打印机 | 不做 |
| 第三方跳过 Edge 确认 | 不做 |
| Spooler 等回退打印 | 禁止 |
| UNCONFIRMED 静默重打 | 禁止 |
| Users / Settings / OAuth2Clients 管理页 | 源码有未挂载占位或孤儿组件，**无菜单入口**，不算交付 |
| 独立运维迁移工具 | 无；启动时 `InitTables` + `migrations.Run`（`schema_migrations`，版本化 SQL） |
| CI / OTA / 多租户计费 | 未成交付门禁 |
| 自签证书信任链 | 不做 |

里程碑与勾选：工作区 `FlyPrint开发计划.md`、`FlyPrint任务清单.md`（完成态以清单「用法」为准）。
